En cas de problèmes techniques

Il se peut que vous rencontriez certains problèmes techniques pour valider les exercices.
Lisez leur solution au moins une fois avant de commencer les exercices.

  • Si certains exercices précédemment validés en vert redeviennent gris et "En attente de code à tester" : c'est qu'il y a une erreur de syntaxe dans votre code. Solution : ouvrez la console développeur pour trouver l'erreur.

    Cliquez ici pour savoir comment ouvrir la console développeur de Chrome

    Sur Windows ou Linux : depuis Chrome, tapez Ctrl + Shift + J ou allez dans le menu (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome) puis allez dans Plus d'outils > Outils de développement.

    Sur MacOS : depuis Chrome, tapez ⌘ + ⌥ + J (Commande + Option + J) ou bien allez dans le menu Afficher > Options pour les développeurs > Console JavaScript.

    Le message d'erreur apparaît en rouge et est en anglais, il faudra le traduire. L'erreur vient du fichier noté sur la droite au numéro de ligne indiqué. Ouvrez ce fichier dans VSCode, rendez-vous à la ligne indiquée et corrigez l'erreur.

  • Si la page web ne répond plus (les clics n'ont aucun effet) ou si l'icône dans l'onglet tourne indéfiniment : c'est qu'il y a une boucle infinie dans votre code. Solution : corrigez votre code puis actualisez le site web.

    Cliquez ici pour voir comment corriger une boucle infinie

    Si vous avez fait une boucle infinie, l'icône de votre onglet va tourner comme ça indéfiniment :

    Voici à quoi peut ressembler une boucle infinie dans du code :

    let poidsClient = prochainClient.mesurerPoids();

    while (poidsClient !== 0) {
      console.log(poidsClient);
    }

    Le problème ici, c'est que la condition de répétition de la boucle poidsClient !== 0 est toujours vraie ! En effet, on ne modifie pas la valeur de poidsClient à l'intérieur de la boucle, par conséquent celle-ci s'exécutera sans jamais s'arrêter : vous avez fait une boucle infinie !

    Voici les étapes à respecter pour se sortir de ce problème :

    1. Dans le menu de Chrome (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome), allez dans Plus d'outils > Gestionnaire de tâches, sélectionnez l'onglet qui pose problème dans la liste puis cliquez sur le bouton Arrêter le processus en bas à droite.
      Vous pouvez également faire un clic droit sur la barre de titre de Chrome pour sélectionner plus rapidement le gestionnaire de tâches.

    2. Corrigez le bogue dans votre code depuis VSCode.
    3. Actualisez à nouveau la page web qui a été précédemment arrêtée en cliquant sur la petite flèche à gauche de la barre d'adresse.

    Attention à la sauvegarde automatique dans VSCode !

    Dans les paramètres, je vous conseille d'utiliser l'option onFocusChange pour la sauvegarde automatique.

    En effet, si vous avez l'option afterDelay de sélectionnée, ça veut dire que VSCode va sauvegarder votre code toutes les X secondes. Le site web va se recharger automatiquement à ce moment là et si votre code actuel génère une boucle infinie, Chrome va planter !

    Pour éviter ce problème, utilisez l'option onFocusChange qui indique à VSCode de sauvegarder votre fichier seulement si vous changez de fenêtre ou d'onglet. Vous pouvez aussi sauvegarder manuellemment avec ⌘ + S sur MacOS ou Ctrl + S sur Windows ou Linux quand vous voulez vérifier que votre code fonctionne.

Structure des exercices

Exercices d'échauffement

Dans un premier temps, vous allez devoir utiliser les méthodes listées ci-dessus une par une, dans l'ordre, pour réaliser les exercices d'échauffement. Vous ferez quelques exercices sur abs(), puis quelques exercices sur ceil(), puis floor() etc.

Même si ces exercices peuvent parfois être un peu rébarbatifs, faites-les au moins une fois. L'objectif est de vous familiariser avec chacune de ces méthodes pour retenir l'utilité principale de celles-ci.

Exercices finaux

Une fois l'échauffement terminé, les exercices suivants vont devenir un peu plus difficiles : vous devrez utiliser plusieurs des méthodes vues précédemment pour répondre à l'énoncé.

Indices

Pour les exercices finaux vous pourrez afficher un indice qui vous indiquera quelles méthodes utiliser pour répondre à l'énoncé. N'affichez l'indice que si vous avez passé au moins 30 min sur un exercice. Le but n'est pas d'aller le plus vite possible mais de faire un maximum d'exercices seul.

L'objectif est de vous apprendre à utiliser les nouveaux outils à votre disposition pour répondre à des exercices de plus en plus complexes.

Vous développerez votre sens logique et votre capacité à assembler des briques de base (les méthodes) entre elles pour répondre à des énoncés complexes.

Fonctionnement des exercices

Tous les exercices fonctionnent sur le même principe : vous devez rédiger une fonction à partir de zéro et celle-ci devra renvoyer (avec le mot-clé return) la solution de l'énoncé.

Exercices finaux

Exercice 0 - Thermomètre endommagé

La sonde de température du robot Persévérance qui a atterit sur Mars en février 2021 a été endommagé. Elle ne retourne plus la température absolue actuelle mais l'écart de température avec la dernière prise de température !

La dernière température absolue relevée était de -54 °C (oui ça caille sur Mars !).

Vous êtes chargé de créer une fonction corrigerTemp(liste) qui transforme une liste de différences de températures en une liste de températures absolues.

En plus de ne plus indiquer la température absolue, les valeurs de différences de température renvoyées ont des décimales inutiles, tronquez-les avant de faire vos calculs !

L'argument liste est un tableau contenant des nombres représentants les différences de températures observées chaque jour sur Mars.

Votre fonction doit renvoyer un tableau avec les températures absolues pour chaque jour, voici un exemple :

  1. Soit liste = [-2.34, -8.12, 5.74, 20.8, -30.82]
  2. La valeur initiale de température est toujours la même : -54 °C
  3. La valeur de retour devra être [-56, -64, -59, -39, -69]
Pensez à bien utiliser la température absolue du jour précédent pour calculer celle du jour d'après.

Indice

Pour tronquer chaque valeur de différence de température il suffit d'utiliser la méthode Math.trunc(). Ensuite il faut parcourir dans une boucle chaque valeur. Utilisez une variable temperaturePrecedente par exemple pour stocker la température absolue du jour d'avant. Il ne reste plus qu'à ajouter la valeur de différence de température à celle-ci puis à ajouter le résultat dans un tableau resultat.

Exercice 1 - Vent solaire

Un vent solaire important provoque une perturbation électromagnétique sur le robot : son capteur de pression atmosphérique déraille complètement !

Parfois il renvoie des valeurs négatives alors qu'il ne devrait renvoyer que des valeurs positives. Par ailleurs, il renvoie des valeurs avec une partie décimale alors qu'elles devraient être arrondies à l'entier le plus proche.

Vous êtes chargé de créer une fonction corrigerPression(liste) qui corrige une liste de valeurs de pression erronées en une liste avec des valeurs correctes.

L'argument liste est un tableau contenant des nombres représentants la pression atmosphérique observées chaque jour sur Mars.

Votre fonction doit renvoyer un tableau avec les pressions corrigées pour chaque jour, voici un exemple :

  1. Soit liste = [824, -840, 812, -822.54, -830.12]
  2. Toutes les valeurs corrigées doivent être positives. Donc par exemple la valeur -840 doit être corrigée pour la valeur 840
  3. Toutes les valeurs doivent être arrondies à l'entier le plus proche. Donc par exemple la valeur -822.54 doit être corrigée pour la valeur 823 : on transforme d'abord en nombre positif puis on arrondit.
  4. La valeur de retour devra être [824, 840, 812, 823, 830]
Indice

Lisez bien l'énoncé, il faut d'abord corriger le signe de la valeur et ensuite corriger l'arrondi. Pour corriger le signe il suffit d'utiliser Math.abs() pour toujours avoir une valeur positive. Pour corriger l'arrondi il suffit d'utiliser Math.round(). La même mécanique de boucle que l'exercice précédent peut être utilisée.
