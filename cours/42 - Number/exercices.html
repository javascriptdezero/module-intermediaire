En cas de problèmes techniques

Il se peut que vous rencontriez certains problèmes techniques pour valider les exercices.
Lisez leur solution au moins une fois avant de commencer les exercices.

  • Si certains exercices précédemment validés en vert redeviennent gris et "En attente de code à tester" : c'est qu'il y a une erreur de syntaxe dans votre code. Solution : ouvrez la console développeur pour trouver l'erreur.

    Cliquez ici pour savoir comment ouvrir la console développeur de Chrome

    Sur Windows ou Linux : depuis Chrome, tapez Ctrl + Shift + J ou allez dans le menu (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome) puis allez dans Plus d'outils > Outils de développement.

    Sur MacOS : depuis Chrome, tapez ⌘ + ⌥ + J (Commande + Option + J) ou bien allez dans le menu Afficher > Options pour les développeurs > Console JavaScript.

    Le message d'erreur apparaît en rouge et est en anglais, il faudra le traduire. L'erreur vient du fichier noté sur la droite au numéro de ligne indiqué. Ouvrez ce fichier dans VSCode, rendez-vous à la ligne indiquée et corrigez l'erreur.

  • Si la page web ne répond plus (les clics n'ont aucun effet) ou si l'icône dans l'onglet tourne indéfiniment : c'est qu'il y a une boucle infinie dans votre code. Solution : corrigez votre code puis actualisez le site web.

    Cliquez ici pour voir comment corriger une boucle infinie

    Si vous avez fait une boucle infinie, l'icône de votre onglet va tourner comme ça indéfiniment :

    Voici à quoi peut ressembler une boucle infinie dans du code :

    let poidsClient = prochainClient.mesurerPoids();

    while (poidsClient !== 0) {
      console.log(poidsClient);
    }

    Le problème ici, c'est que la condition de répétition de la boucle poidsClient !== 0 est toujours vraie ! En effet, on ne modifie pas la valeur de poidsClient à l'intérieur de la boucle, par conséquent celle-ci s'exécutera sans jamais s'arrêter : vous avez fait une boucle infinie !

    Voici les étapes à respecter pour se sortir de ce problème :

    1. Dans le menu de Chrome (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome), allez dans Plus d'outils > Gestionnaire de tâches, sélectionnez l'onglet qui pose problème dans la liste puis cliquez sur le bouton Arrêter le processus en bas à droite.
      Vous pouvez également faire un clic droit sur la barre de titre de Chrome pour sélectionner plus rapidement le gestionnaire de tâches.

    2. Corrigez le bogue dans votre code depuis VSCode.
    3. Actualisez à nouveau la page web qui a été précédemment arrêtée en cliquant sur la petite flèche à gauche de la barre d'adresse.

    Attention à la sauvegarde automatique dans VSCode !

    Dans les paramètres, je vous conseille d'utiliser l'option onFocusChange pour la sauvegarde automatique.

    En effet, si vous avez l'option afterDelay de sélectionnée, ça veut dire que VSCode va sauvegarder votre code toutes les X secondes. Le site web va se recharger automatiquement à ce moment là et si votre code actuel génère une boucle infinie, Chrome va planter !

    Pour éviter ce problème, utilisez l'option onFocusChange qui indique à VSCode de sauvegarder votre fichier seulement si vous changez de fenêtre ou d'onglet. Vous pouvez aussi sauvegarder manuellemment avec ⌘ + S sur MacOS ou Ctrl + S sur Windows ou Linux quand vous voulez vérifier que votre code fonctionne.

Exercice 0

Créez une fonction isNaN0(valeur) qui renvoie true si la valeur vaut NaN et false sinon.

Exemples :

  • isNaN0("JavaScript") renverra false.
  • isNaN0("NaN") renverra false.
  • isNaN0(NaN) renverra true.

Exercice 1

Exemples :

  • isFinite0("0") renverra false.
  • isFinite0(0) renverra true.
  • isFinite0(5/2) renverra true.

  • isFinite0(NaN) renverra false.
  • isFinite0(-Infinity) renverra false.

Exercices finaux

Exercice 0 - Décodeur

Créez une fonction decodeur(secret) qui décode le message secret. Celui-ci est une chaîne de caractères composée uniquement de 0 et de 1 qui représentent le message original.

Par exemple, supposons que secret vaut 0110011001110010.

Pour décoder ce secret, vous devrez suivre les étapes suivantes :

  1. Découpez la chaîne en blocs de 8 chiffres consécutifs. Chaque bloc de 8 chiffres représente un nombre au format binaire.
  2. Convertissez ce nombre en décimal. Ici, le 1er bloc est 01100110 et vaut 102. Le 2ème bloc est 01110010 et vaut 114.
  3. Convertissez chaque nombre en son caractère associé selon la table de correspondances ASCII. La méthode String.fromCharCode(nombre) vous permet de faire ça facilement !
  4. Le nombre 102 correspond à la lettre f.
  5. Le nombre 114 correspond à la lettre r.
  6. Le message clair était donc fr.

Il est garanti que la longueur de secret est toujours un multiple de 8.

Exemples :

  • decodeur("0110011001110010") renverra "fr".
  • decodeur("01001010011000010111011001100001010100110110001101110010011010010111000001110100") renverra "539".
Indice

Pour découpez chaque bloc de 8 chiffres, utilisez une boucle avec slice(). Faites les conversions nécessaires avec parseInt() et la base qui va bien puis String.fromCharCode().

Exercice 1 - Encodeur

Créez une fonction encodeur(message) qui encode le message sous la forme du secret de l'exercice précédent.

Par exemple, supposons que le message à encoder soit JavaScript.

Pour encoder ce message, vous devrez suivre les étapes suivantes :

  1. Pour chaque caractère dans le message, convertissez-le en son nombre décimal équivalent selon la table de correspondances ASCII. La méthode charCodeAt() vous permet de faire ça facilement !
  2. La lettre J correspond au nombre 74
  3. Convertissez chaque nombre en son équivalent binaire grâce à la méthode toString(). Le nombre 74 devient 01001010. Attention, la représentation binaire doit être sur 8 chiffres, il faudra compléter avec des 0 au début si besoin.
  4. Concaténez le tout en une seule chaîne de caractères qu'il faut renvoyer !

Exemples :

  • encodeur("fr") renverra "0110011001110010".
  • decodeur("JavaScript") renverra "01001010011000010111011001100001010100110110001101110010011010010111000001110100"
Indice

Lisez chaque caractère du message avec une boucle et récupérez le nombre correspondant à ce caractère avec charCodeAt(). Faites les conversions nécessaires avec toString() et la base qui va bien puis complétez avec les 0 nécessaires devant la représentation binaire grâce à padStart().

Exercice 2 - Panier de fruits (difficile)

Créez une fonction totalPanier(commandes) qui retourne une chaîne de caractères représentant la somme de toutes les commandes passées par plusieurs clients.

Les commandes sont au format CSV, sous la forme d'une chaîne de caractères, par exemple : 10 pommes,20 cerises,35 pommes,12 bananes,30 cerises,6 bananes.

La fonction devra calculer la somme totale de fruits à vendre et retourner cette somme dans une chaîne de caractères comme ceci : 45 pommes,50 cerises,18 bananes.

L'ordre des fruits devra être le même que celui rencontré dans les commandes.

Pour simplifier l'exercice, il est garanti que le nom du fruit est toujours écrit au pluriel, par exemple, dans la commande, il peut être écrit 0 bananes,1 bananes.

Si la somme des commandes d'un certain fruit vaut 0, on ne l'indique pas du tout dans le total.

Exemples :

  • totalPanier("10 pommes,20 cerises,35 pommes,12 bananes,30 cerises,6 bananes") renverra 45 pommes,50 cerises,18 bananes
  • totalPanier("2 melons,12 oranges,35 pommes,1 melons,0 bananes,5 oranges,0 bananes,30 cerises") renverra 3 melons,17 oranges,35 pommes,30 cerises
Indice 1

L'astuce principale à exploiter pour cet exercice est d'utiliser un objet JavaScript (nommons-le sommeCommandes par exemple) pour stocker la somme des quantités de chaque fruit sous la forme { "pommes": 20, "cerises": 35 }.

À chaque nouveau fruit qu'on rencontre on ajoute une nouvelle propriété dans cet objet qui aura pour nom le nom du fruit et pour valeur la quantité demandée.

Si on a déjà rencontré ce fruit, on ajoute la nouvelle quantité à la quantité déjà existante dans sommeCommandes. Par exemple si on avait { "pommes": 10 } et qu'on lit 7 pommes dans la liste des commandes, il faut modifier l'objet pour avoir { "pommes": 17 } et ainsi de suite.

Utilisez la méthode split() pour découper la chaîne commandes et en extraire les quantités et le nom de chaque fruit facilement. Ensuite faites une boucle pour récupérer chaque commande de fruits séparément et alimenter l'objet sommeCommandes.

Dans un premier temps, essayez de construire cet objet sommeCommandes correctement, sans vous soucier du reste.

Ensuite il faudra trouver un moyen d'afficher dans l'ordre qu'il faut le contenu de sommeCommandes. Essayez de votre côté puis lisez l'indice 2 si vous n'y arrivez pas.

Indice 2

Pour retenir l'ordre des fruits, on peut utiliser un tableau ordreDesFruits qui contiendra la liste des noms de chaque fruit qu'on a lu dans commandes. À chaque fois qu'on lit un nouveau nom de fruit, on vérifie qu'il n'est pas déjà dans ordreDesFruits grâce à la méthode includes(). Si ce n'est pas le cas alors on l'ajoute à ordreDesFruits avec la méthode push().

À la fin on obtient dans ce tableau la liste des noms des fruits comme ceci : ["pommes", "cerises", "oranges"]. Il ne reste plus qu'à lire chaque nom de fruit dans ce tableau avec une boucle et à utiliser ce nom de fruit avec sommeCommandes pour afficher la quantité et le nom du fruit dans le bon ordre.

Cet exercice est très difficile. Vous aurez besoin d'utiliser les objets et les tableaux pour y arriver. N'hésitez pas à passer plusieurs heures sur celui-ci, ne vous découragez pas.
