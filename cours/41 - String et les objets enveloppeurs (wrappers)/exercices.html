En cas de problèmes techniques

Il se peut que vous rencontriez certains problèmes techniques pour valider les exercices.
Lisez leur solution au moins une fois avant de commencer les exercices.

  • Si certains exercices précédemment validés en vert redeviennent gris et "En attente de code à tester" : c'est qu'il y a une erreur de syntaxe dans votre code. Solution : ouvrez la console développeur pour trouver l'erreur.

    Cliquez ici pour savoir comment ouvrir la console développeur de Chrome

    Sur Windows ou Linux : depuis Chrome, tapez Ctrl + Shift + J ou allez dans le menu (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome) puis allez dans Plus d'outils > Outils de développement.

    Sur MacOS : depuis Chrome, tapez ⌘ + ⌥ + J (Commande + Option + J) ou bien allez dans le menu Afficher > Options pour les développeurs > Console JavaScript.

    Le message d'erreur apparaît en rouge et est en anglais, il faudra le traduire. L'erreur vient du fichier noté sur la droite au numéro de ligne indiqué. Ouvrez ce fichier dans VSCode, rendez-vous à la ligne indiquée et corrigez l'erreur.

  • Si la page web ne répond plus (les clics n'ont aucun effet) ou si l'icône dans l'onglet tourne indéfiniment : c'est qu'il y a une boucle infinie dans votre code. Solution : corrigez votre code puis actualisez le site web.

    Cliquez ici pour voir comment corriger une boucle infinie

    Si vous avez fait une boucle infinie, l'icône de votre onglet va tourner comme ça indéfiniment :

    Voici à quoi peut ressembler une boucle infinie dans du code :

    let poidsClient = prochainClient.mesurerPoids();

    while (poidsClient !== 0) {
      console.log(poidsClient);
    }

    Le problème ici, c'est que la condition de répétition de la boucle poidsClient !== 0 est toujours vraie ! En effet, on ne modifie pas la valeur de poidsClient à l'intérieur de la boucle, par conséquent celle-ci s'exécutera sans jamais s'arrêter : vous avez fait une boucle infinie !

    Voici les étapes à respecter pour se sortir de ce problème :

    1. Dans le menu de Chrome (les 3 petits points verticaux en haut à droite de la fenêtre de Chrome), allez dans Plus d'outils > Gestionnaire de tâches, sélectionnez l'onglet qui pose problème dans la liste puis cliquez sur le bouton Arrêter le processus en bas à droite.
      Vous pouvez également faire un clic droit sur la barre de titre de Chrome pour sélectionner plus rapidement le gestionnaire de tâches.

    2. Corrigez le bogue dans votre code depuis VSCode.
    3. Actualisez à nouveau la page web qui a été précédemment arrêtée en cliquant sur la petite flèche à gauche de la barre d'adresse.

    Attention à la sauvegarde automatique dans VSCode !

    Dans les paramètres, je vous conseille d'utiliser l'option onFocusChange pour la sauvegarde automatique.

    En effet, si vous avez l'option afterDelay de sélectionnée, ça veut dire que VSCode va sauvegarder votre code toutes les X secondes. Le site web va se recharger automatiquement à ce moment là et si votre code actuel génère une boucle infinie, Chrome va planter !

    Pour éviter ce problème, utilisez l'option onFocusChange qui indique à VSCode de sauvegarder votre fichier seulement si vous changez de fenêtre ou d'onglet. Vous pouvez aussi sauvegarder manuellemment avec ⌘ + S sur MacOS ou Ctrl + S sur Windows ou Linux quand vous voulez vérifier que votre code fonctionne.

  • Si vous souhaitez déboguer votre code, utilisez le débogueur de Chrome comme je l'explique dans cette vidéo.

  • Si vous êtes bloqué et qu'aucune solution mentionnée ne fonctionne, envoyez-moi un email sur jeremy@javascriptdezero.com en précisant le numéro de la leçon sur laquelle vous travaillez et le numéro de l'exercice qui pose problème.

Structure des exercices

Exercices d'échauffement

Dans un premier temps, vous allez devoir utiliser les méthodes listées ci-dessus une par une, dans l'ordre, pour réaliser les exercices d'échauffement. Vous ferez quelques exercices sur charAt(), puis quelques exercices sur concat(), puis includes() etc.

Même si ces exercices peuvent parfois être un peu rébarbatifs, faites-les au moins une fois. L'objectif est de vous familiariser avec chacune de ces méthodes pour retenir l'utilité principale de celles-ci.

Exercices finaux

Une fois l'échauffement terminé, les exercices suivants vont devenir un peu plus difficiles : vous devrez utiliser plusieurs des méthodes vues précédemment pour répondre à l'énoncé.

Indices

Pour les exercices finaux vous pourrez afficher un indice qui vous indiquera quelles méthodes utiliser pour répondre à l'énoncé. N'affichez l'indice que si vous avez passé au moins 30 min sur un exercice. Le but n'est pas d'aller le plus vite possible mais de faire un maximum d'exercices seul.

L'objectif est de vous apprendre à utiliser les nouveaux outils à votre disposition (les méthodes des Strings) pour répondre à des exercices de plus en plus complexes.

Vous développerez votre sens logique et votre capacité à assembler des briques de base (les méthodes) entre elles pour répondre à des énoncés complexes.

Fonctionnement des exercices

Tous les exercices fonctionnent sur le même principe : vous devez rédiger une fonction à partir de zéro et celle-ci devra renvoyer (avec le mot-clé return) la solution de l'énoncé.

Exercice 0

Créez une fonction charAt0(chaine, index) qui renvoie le caractère à l'index index de la chaîne chaine.

Il est garanti que la longueur de chaine est toujours supérieure à index.

Exemple : charAt0("JavaScript", 4) renverra "S".

La méthode charAt() est équivalente à l'utilisation de la notation avec les crochets [].

Exercice 1

Créez une fonction concat0(chaine1, chaine2) qui renvoie la concaténation de chaine1 et chaine2 dans cet ordre.

Exemple : concat0("Java", "Script") renverra "JavaScript".

La méthode concat() est équivalente à l'utilisation de l'opérateur + avec la notation chaine1 + chaine2.

Exercice 2

Créez une fonction concat1(chaine1, chaine2, chaine3) qui renvoie la concaténation de chaine1 et chaine2 et chaine3 dans cet ordre.

Exemple : concat1("Les Strings", " c'est ", "facile !") renverra "Les Strings c'est facile !".

Exercice 3

Créez une fonction concat2(chaine1, chaine2, chaine3) qui renvoie la concaténation de chaine3 et chaine2 et chaine1 dans cet ordre.

Exemple : concat2("Yoda", " parle ", "Comme ça") renverra "Comme ça parle Yoda".

Exercice 4

Exemples :

  • includes0("JavaScript", "S") renverra true.
  • includes0("JavaScript", "s") renverra false.

Exercice 6

Exemples :

  • includes2("JavaScript", "t") renverra true.
  • includes2("JavaScript", "i") renverra false.

Contrairement à la méthode Array.prototype.includes() (sur les tableaux donc !), il n'est pas possible d'utiliser d'index négatif avec la méthode String.prototype.includes() ! Mais vous pouvez facilement contourner cette limitation en utilisant chaine.length...

Exercices finaux

Exercice 0 — Vérification de jauge

Créez une fonction verifierJauge(jauge) qui renverra true si la jauge est remplie à au moins 50% et false sinon. La jauge est une chaîne de caractères composée d'espaces et du caractère # répété plusieurs fois.

Exemples :

  • verifierJauge("####  ")
    renverra true
  • verifierJauge("###   ")
    renverra true
  • verifierJauge("##    ")
    renverra false
  • verifierJauge("##            ")
    renverra false
  • verifierJauge("#######  ")
    renverra true
Indice

Utilisez lastIndexOf() et la valeur.length pour calculer le taux de remplissage de la jauge.

Exercice 1 — Création de jauge

Créez une fonction creerJauge(pourcentage) qui renverra une chaîne de caractères de 10 caractères représentant le pourcentage avec des # complété par des espaces.

Le pourcentage est forcément un multiple de 10.

Exemples :

  • creerJauge(0)
    renverra
    (10 espaces)
  • creerJauge(10)
    renverra 
    #         
  • creerJauge(50)
    renverra 
    #####     
  • creerJauge(80)
    renverra 
    ########  
  • creerJauge(100)
    renverra 
    ##########
Indice

Utilisez la fonction repeat() et padEnd() pour générer la jauge facilement.

Exercice 2 — Compteur de voyelles

Créez une fonction compteurVoyelles(phrase) qui renvoie le nombre de voyelles (a,e,i,o,u,y) contenues dans la phrase qui sera toujours écrite en minuscules.

Exemples :

  • compteurVoyelles("apprendre pour progresser") renverra 8.
  • compteurVoyelles("la cle, c'est de perseverer") renverra 8.
  • compteurVoyelles("un peu chaque jour pour memoriser") renverra 14.
Indice

Lisez chaque lettre pour voir si c'est une voyelle et incrémentez un compteur. Pour vérifier qu'une lettre est une voyelle, on peut utiliser l'astuce "aeiouy".includes(lettre) !

Exercice 3 — Compteur de consonnes

Créez une fonction compteurConsonnes(phrase) qui renvoie le nombre de consonnes, soit toutes les lettres sauf a,e,i,o,u,y contenues dans la phrase qui sera toujours écrite en minuscules.

Exemples :

  • compteurConsonnes("apprendre pour progresser") renverra 15.
  • compteurConsonnes("la cle, c'est de perseverer") renverra 13.
  • compteurConsonnes("un peu chaque jour pour memoriser") renverra 14.
Indice

Même chose que l'exercice précédent mais il suffit de changer la liste des voyelles pour les consonnes !

Exercice 4 — Kebab case

Créez une fonction kebabCase(chaine) qui renvoie une chaîne de caractères contenant tous les mots écrits dans la chaine au format kebab-case.

Pour rappel, un nom au format kebab-case est entièrement en minuscules et les espaces sont remplacés par des tirets.

Exemples :

  • kebabCase("liste noeuds HTML") renverra liste-noeuds-html.
  • kebabCase("Liste des items selection") renverra liste-des-items-selection.
Indice

Utilisez d'abord toLowerCase() puis replaceAll() !

Exercice 5 — Camel case

Créez une fonction camelCase(chaine) qui renvoie une chaîne de caractères contenant tous les mots écrits dans la chaine au format camelCase et concaténés ensemble.

Pour rappel, un nom au format camelCase commence par une minuscule puis chaque première lettre de chaque mot est en majuscules.

Exemples :

  • camelCase("liste noeuds HTML") renverra listeNoeudsHtml.
  • camelCase("Liste des items selection") renverra listeDesItemsSelection.
Indice

Commencez par découper la chaîne sous la forme d'un tableau de mots avec split(), puis pour chaque mot sauf le 1er, extraire la 1ère lettre, la mettre en majuscule avec toUpperCase() et utilisez slice() pour récupérer le reste du mot. Concaténez tout ça à une chaîne de résultat.

Exercice 6 — Tri de mots

Créez une fonction triDeMots(liste) qui renvoie une chaîne de caractères contenant uniquement les mots qui se trouvent dans la liste commençant et terminant par une voyelle a,e,i,o,u,y.

Exemples :

  • triDeMots("air,eau,feu,terre,oui,non,ame") renverra eau,oui,ame.
  • triDeMots("exercice,courage,volley,penny,amy") renverra exercice,amy.
Indice

Dans un premier temps il faut utiliser Array.prototype.split() pour créer un tableau contenant chaque mot de la phrase. À partir de là, on boucle sur chaque mot, on vérifie qu'il respecte les conditions puis on le concatène dans une chaîne de caractères de résultat.

Exercice 7 — Légumes

Créez une fonction legumes(liste) qui renvoie une chaîne de caractères contenant uniquement les légumes qui se trouvent dans la liste.

La liste des légumes possibles : carotte,haricot,chou,aubergine

Exemples :

  • legumes("carotte,pomme,cerise,haricot,chou,poire") renverra carotte,haricot,chou.
  • legumes("pomme,aubergine,chou,haricot,pomme,poire") renverra aubergine,chou,haricot.
Indice

Même chose que l'exercice précédent, on pourra également utiliser l'astuce avec "carotte,haricot,chou,aubergine".includes(mot) pour détecter si le mot est un légume ou pas.

Exercice 8 — Décodeur

Créez une fonction decodeur(secret) qui renvoie une chaîne de caractères contenant le message clair contenu dans secret.

Le message clair est contenu entre 2 séquences ## et il faut remplacer tous les - par des espaces.

Il est garanti qu'il n'existe qu'un seul message clair dans le secret.

Ensuite il faut convertir chaque chiffre suivant par une lettre :

  • 1 → i
  • 3 → e
  • 4 → a
  • 5 → s
  • 8 → b
  • 0 → o

Exemples :

  • decodeur("fsihgE24TfsE#Efsf##L3-J4v45cr1pt,-c'35t-834u-!##fSH345Zgze") renverra Le Javascript, c'est beau !.
Indice

Utilisez indexOf() et lastIndexOf() pour extraire le contenu intéressant avec slice(). Ensuite utilisez replaceAll() pour déchiffrer le message !

Exercice 9 — Extraction de noms

Créez une fonction extraireNoms(liste) qui renvoie une chaîne de caractères contenant tous les noms écrits en majuscules de la liste et séparés par une virgule, dans l'ordre où ils apparaissent dans liste.

La liste est une chaîne de caractère au format CSV nom:redocemia;prénom:tom,nom:dusse;prénom:jean-claude

Attention l'ordre du nom et du prénom n'est pas garanti, liste pourrait également contenir : prénom:tom;nom:redocemia,nom:dusse;prénom:jean-claude

Exemples :

  • extraireNoms("nom:redocemia;prénom:tom,nom:dusse;prénom:jean-claude") renverra REDOCEMIA,DUSSE.
  • extraireNoms("prénom:tom;nom:redocemia,nom:dusse;prénom:jean-claude") renverra REDOCEMIA,DUSSE.
Indice

Découpez avec split() pour extraire chaque couple "nom;prénom" puis redécoupez avec split() et cherchez si le préfixe nom: est présent avec startsWith(). Si oui, extrayez le nom, mettez-le en majuscules avec toUpperCase() et ajouter ça au résultat !

Exercice 10 — Tableur (difficile)

Créez une fonction tableur(liste) qui renvoie une chaîne de caractères contenant une liste des prénoms et noms contenus dans liste alignés sur la droite comme dans un tableur. Un en-tête sera présent avec les mots Prénom et NOM. Les colonnes seront séparées de 3 espaces.

Pour créer un retour à la ligne dans la chaîne de caractères renvoyée, utilisez le caractère d'échappement \n.

Le contenu de la liste respectera toujours l'ordre Prénom NOM,Prénom NOM, etc..

Exemples :

  • aligner("Tom REDOCEMIA,Jean-Claude DUSSE") renverra 
         Prénom         NOM
        Tom   REDOCEMIA
Jean-Claude       DUSSE
  • aligner("Linus TORVALDS,Bill GATES,Ada LOVELACE") renverra 
    Prénom        NOM
 Linus   TORVALDS
  Bill      GATES
   Ada   LOVELACE
Indice

Prenez votre temps, décortiquez chaque étape tranquillement. Utilisez padStart() pour créer facilement les espaces dont vous aurez besoin pour le formatage. Notez bien qu'il faudra dans un premier temps calculer la longueur max des prénoms et des noms car ce sont ces longueurs qui vont déterminer les largeurs des colonnes Prénom et NOM. Utilisez \n pour faire des nouvelles lignes dans la chaîne de caractères que vous devez renvoyer. Bon courage !

Exercice 11 — Ville ou pays ? (difficile)

Créez une fonction villeOuPays(liste) qui renvoie une chaîne de caractères contenant tous les mots écrits dans la liste en les préfixant de VILLE: ou PAYS: en fonction de si le mot est le nom d'une ville ou d'un pays. Cette fonction corrigera également le formatage des noms : un nom correctement formaté est écrit tout en minuscules avec seule sa première lettre en majuscule.

Les villes possibles sont : Paris, Londres, Rome, Berlin.

Les pays possibles sont : France, Royaume-Uni, Italie, Allemagne.

Le contenu de liste n'est pas très bien formaté. Il y a parfois des espaces en trop et des majuscules un peu n'importe où ! Il faudra corriger tout ça au passage !

C'est un problème classique ces histoires de formatage... Une astuce consiste à convertir au préalable les 2 chaînes qu'on veut comparer en minuscules (ou majuscules). Pour comparer France et franCE on met tout en minuscules et on compare pour voir que ça correspond !

Exemples :

  • villeOuPays("parIS, royaume-UNI,ITALIE ,berlin,franCE") renverra VILLE:Paris,PAYS:Royaume-Uni,PAYS:Italie,VILLE:Berlin,PAYS:France.
  • villeOuPays("roME, AlleMagne , londres, BERLIN") renverra VILLE:Rome,PAYS:Allemagne,VILLE:Londres,VILLE:Berlin.
Indice

Prenez votre temps, décortiquez chaque étape tranquillement. Utilisez 2 tableaux de référence pour stocker les noms des villes et pays. Faites vos comparaisons avec toutes les valeurs en minuscules (grâce à toLowerCase()) pour éviter les soucis de formatage des lettres. Ensuite concaténez le préfixe qui va bien avec le nom récupéré dans les tableaux de référence et mettez tout ça dans le résultat. Bon courage !
